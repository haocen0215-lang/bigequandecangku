## 步骤 3：定义主存档结构（chat 变量）

**核心原则：** 主存档放 chat 变量（`getVariables({ type: 'chat' })`），不依赖 MVU 楼层变量。MVU 仅用于楼层快照和兼容性展示。

**需要定义的内容：**

### 3.1 存档顶层结构
```typescript
interface GameSave {
  schemaVersion: number;        // 用于迁移
  savedAt: number;              // Date.now()
  player: PlayerData;           // <user> 角色数据
  tavern: TavernData;           // 酒馆数据（含所有区域+风格）
  staff: StaffMember[];         // 员工
  industries: IndustryMap;      // 产业
  factions: FactionStandingMap; // 势力关系
  inventory: InventoryMap;      // 库存（食材/调料/成品/其他）
  menu: MenuItem[];             // 菜单
  crops: CropPlotMap;           // 种田（地块列表）
  animals: AnimalMap;           // 养殖
  relations: RelationMap;       // 后宫关系
  adventure: AdventureData;     // 冒险/战斗
  world: WorldState;            // 世界状态（时间/位置/天气/季节）
  ledger: LedgerData;           // 账本
  chronicle: ChronicleEntry[];  // 编年史
}
```

### 3.2 各模块简略结构

- **player**: 种族/背景文字/出身描述/精力/HP/经济/穿着/位置/烹饪等级+次数
- **tavern**: 名称/所在城市/风格描述/区域列表（每个区域=设施标签[]+状态+风格描述）/分店列表
- **staff**: [{id, name, race, role, wage, loyalty, description}]
- **industries**: {[name]: {type, status, invested, income, incomePeriod, note}}
- **factions**: {[factionId]: {standing, level, lastEvent}}
- **inventory**: {[itemName]: {qty, unit, category, priceCopper, flavorTags, cookMethod}}
- **crops**: {[plotId]: {cropType, plantedAt, stage, health, events}}
- **animals**: {[animalType]: {count, fed, products}}
- **relations**: {[charId]: {stage, description, favor, ...}}
- **world**: {date/time/season/weather/location}
- **ledger**: {daysOpen, totalCustomers, totalIncome, assets}
- **adventure**: {party, quests, combatState, explorationData}

## 步骤 6：写 Engine 规则（最核心）

### 6.1 `src/tavern_status/engine/` 目录

```
engine/
  tick.ts            # 时间推进（现实时间→游戏内时间）
  cooking.ts         # 烹饪引擎（食材搭配冲突检测、标签推导、定价）
  farming.ts         # 种田引擎（生长周期、季节影响、随机事件）
  business.ts        # 经营结算（客流、售出、库存扣减、收支、声望）
  dispatch.ts        # 统一 Action 派发入口
  actionTypes.ts     # Action 类型定义
```

### 6.2 烹饪引擎（参考普利莫迪亚菜品生成引擎）
- 食材角色分配（主料/副主料/配菜/配味/芳香底味/未分类物品）
- 烹饪方式选择（等级解锁约束）
- 菜名生成
- 搭配冲突检测（4 级：无/轻微/严重/灾难）
- 标签推导（风味+口感+气味）
- 等级修正
- 定价公式

### 6.3 Tick 引擎
- 游戏内时间推进
- 作物生长（按游戏时间）
- 发酵/陈酿进度
- 员工工资日期
- 产业收入周期

### 6.4 经营结算引擎
参考普利莫迪亚 `🧩经营快进引擎.txt`
- 客流判定（声望/季节/天气/星期）
- 售出分配
- 库存扣减
- 收支结算
- 声望变化

## 步骤 7：写 Dispatcher 动作入口

### 7.1 Action 类型

```typescript
type GameAction =
  | { type: 'COOK'; mode: 'immediate' | 'batch'; ingredients: string[]; method?: string; count?: number }
  | { type: 'SERVE'; customerId?: string }
  | { type: 'PLANT_CROP'; plotId: string; cropType: string }
  | { type: 'HARVEST'; plotId: string }
  | { type: 'BUILD'; region: string; facilities: string[]; style?: string }
  | { type: 'HIRE_STAFF'; staffId: string }
  | { type: 'FIRE_STAFF'; staffId: string }
  | { type: 'START_INDUSTRY'; name: string; type: string; investment: number }
  | { type: 'TRAVEL'; destinationId: string }
  | { type: 'ADVENTURE_ENTER'; layer: number }
  | { type: 'COMBAT_ACTION'; skillId: string; targetId: string }
  | { type: 'TICK'; hours: number }
  | { type: 'OPEN_FOR_BUSINESS'; hours: number }
  // ... 更多
```

### 7.2 Dispatcher 实现

一个纯函数：`dispatchAction(save: GameSave, action: GameAction): ActionResponse`

- 校验合法性
- 调用对应 engine 函数
- 返回 ActionResponse（ok/bool + tone + message + shouldAskAI + logs）

## 步骤 10：写前端壳和主要界面

这是**改造 `index.html` + `script.js`** 这一步。把当前纯静态展示改造成连接 engine 的客户端。

### 10.1 入口改造（script.js）

当前 `script.js` 需要改成：
1. 加载时读存档
2. 根据楼层数判断：0 → 开局流程 / >0 → 游戏界面
3. dispatch action 的入口
4. 事件监听（MESSAGE_UPDATED → 刷新数据）

### 10.2 各页面增加交互面板

| 页面 | 当前 | 需要加的交互 |
|---|---|---|
| **剧情** | 静态 HUD + 硬编码故事文本 | 接入 AI 回复流式显示、对话记录 |
| **酒馆** | 区域展示全部静态 | 区域管理（扩建/升级/改风格按钮）、经营快进结算显示、员工面板、产业面板 |
| **库存** | 加选/减选+输出区文本 | 真正扣库存、做成菜时真正调用 engine |
| **购物** | 购物篮+输出区文本 | 真正扣钱+加库存 |
| **账本** | 从硬编码 data-* 读 | 从存档读、结算后实时更新 |
| **玩家档案** | 静态数据 data-* | 从存档读、升级显示、出身背景 |
| **角色** | 3 个硬编码 NPC | 从存档关系数据读、可互动按钮 |
| **地图** | 只读 80+ 节点 | 移动操作、旅行时间、分店选择 |
| **输出区** | 放文字的面板 | 保留但不再作为唯一交互出口 |

### 10.3 新增：开局交互
