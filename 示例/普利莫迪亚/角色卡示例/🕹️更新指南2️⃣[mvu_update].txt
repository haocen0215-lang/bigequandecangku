# 变量更新指南

== 先判断本轮应检查哪些方面 ==

必查: [世界状态, user.穿着位置]

条件检查:
  酒馆经营: 做菜、采购食材、客人反应、查看库存或菜单; 联动→user.经济
  user.精力与战斗: 体力动作（锻炼/劈柴/搬运等）、亲密场景、射精事件、吃喝恢复、受到物理伤害、恢复调养
  user.经济: 买卖交易、采购食材、菜品出售收入、支付工资、任何收支事件; 联动→酒馆经营

  # 配角管理原则：
  # 新配角登场/进入场景 → 在配角字典下新建完整条目
  # 配角退场/离开场景 → 从配角字典中完全删除
  # 以下对每个【当前在场】的配角分别检查:
  配角.${配角名}.位置穿着: 配角移动、换衣、亲密场景每轮自动检查穿着
  配角.${配角名}.身体心理: 亲密场景每轮自动；日常中配角情绪或身体有明显变化时; 联动→配角.${配角名}.关系资金
  配角.${配角名}.关系资金: 与配角发生关系质变事件、配角个人收支、关系定义变化; 联动→配角.${配角名}.身体心理
  配角.${配角名}.性相关: 发生性行为、首次使用新体位

== 再判断每个方面具体有哪些变量需要更新 ==

世界状态: 关注时间流逝幅度——场景内对话推进分钟级，场景切换推进小时级，跳过空白推进天级。日推进时检查星期和月份名是否需要联动更新。季节变化时注意食材可用性联动。

酒馆经营: 做菜事件→累计做菜次数+1，检查是否触发升级门槛（15→帮厨/45→厨师/100→主厨/200→大师傅），新菜名加入菜单，库存对应食材-1。采购→库存+N，经济扣款。客人反应→声望增减。

user.精力与战斗: 三个子系统分别检查——
  精力: 当前值消耗/恢复（动作强度决定消耗速率，吃喝恢复量由菜品质量决定，睡眠→全满）。成长: 主动锻炼→最大值少量增长，长期高强度亲密→最大值微量增长。检查百分比是否跨越分段阈值（充沛/消耗/疲惫/极限/脱力）。0=强制结束。
  射精: 每次射精→当前次数+1且精力额外大额扣除（扣除量随当日次数递增）。达到上限→当日不可再射精。睡眠→当前次数归零。成长: 亲密经验累积→每日上限缓慢增长。
  HP: 受伤→当前值下降。恢复→自然每天少量，药剂/被照顾加速。成长: 受伤后最大值微量增长。0=强制卧床。

user.经济: 精确记录每笔收支——菜品售价收入、食材采购支出、工资支出、其他买卖。注意货币换算（100铜=1银，10银=1金）。

user.穿着位置: 位置——user当前在哪，移动时更新。穿着——按槽位检查是换了什么衣物（内容替换+状态重置为完整），还是状态推进（完整→微乱→半脱→脱下）。注意遮挡链：下装状态影响内裤可观测性。

# 针对每个在场配角:
配角.${配角名}.位置穿着: 位置——配角当前在哪。穿着——7个槽位各自独立检查，注意遮挡链（外层→上装→内衣；下装→内裤）。身上痕迹——亲密后新增或时间流逝/洗澡后消退。

配角.${配角名}.身体心理: 身体——各部位独立更新，受穿着遮挡约束。焦点转移原则: user停止对某部位的直接互动时，必须主动将该部位描写降温（平滑过渡为余韵/自然状态）。心理——一句话当前内心。精力——根据动作强度/持续时间判断档位变化（充沛→正常→疲惫→脱力），休息/吃喝/睡觉恢复。膀胱——时间累积（每小时+3~5）+饮品摄入（一次+15~25），≥50行为表现变化，100触发失禁，上厕所→归零。

配角.${配角名}.关系资金: 亲密度——是否发生不可回退的认知质变（+1或-1），升降档时必须重写亲密度描述。与user关系——双方共识的关系名义是否变化。资金——配角个人收支。

配角.${配角名}.性相关: 性行为次数+1（完成一次完整性交时），新体位加入体位经历。

== 一些通用的判断模式 ==
身体: 伤害/恢复/消耗/成长（最大值增长）
心理: 情绪/决策/领悟/被看穿
关系: 互动/冲突/态度质变/共识变化
进程: 里程碑/阶段转换/门槛达成
资源: 获取/消耗/交易/库存增减
时间: 流逝幅度/日期推进/季节联动
