<WORLD_generative_rules_定居点生成器>
# 名称: 定居点生成器
# 用途: 用于创建普里莫迪亚大陆上的城市、城镇、村庄条目。独有设施、行业地点、命名风格等由AI根据世界书中已有的种族/信仰/文化/产业设定自行推导。同一模板也可在运行时用于AI临时生成旅途中经过的小地方。
# 本生成器分为基础版和扩展包。基础版用于创建符合种族典型印象的标准城市。扩展包在需要创造反直觉/高个性城市时加载。
# 为同一种族创建多个定居点时，每个定居点的"主标签+核心地理类型"组合不得与该种族已有的定居点重复（性质可相同）。如果最典型的组合已经被用了，就必须选其他组合——然后思考这个种族用自己的方式怎么做这件事。

# ═══ 定居点条目模板 ═══

定居点条目: # 类型: obj | 创作: 自由
  定居点名: ${名称} # 类型: str | 创作: 自由
  所属: ${种族/国家/邦} # 类型: str | 创作: 选择 | 来源: 必须从世界蓝图中已有的种族/国家/邦选择
  性质: ${城市/城镇/村庄} # 类型: str | 创作: 选择 | 来源: 必须从枚举值[城市,城镇,村庄]选择
  功能类型:
    主标签: ${类型} # 类型: str | 创作: 选择 | 来源: 必须从'功能类型池'选择
    副标签: # 可扩展数组 | 创作: 选择 | 来源: 必须从'功能类型池'选择，0-2个
      - "$__META_EXTENSIBLE__$"
      - ${类型}
  地理位置:
    核心类型: ${类型} # 类型: str | 创作: 选择 | 来源: 必须从'地理位置核心类型池'选择
    特殊地形: ${标签或"无"} # 大部分城市为"无"。仅当地形确实无法用核心类型单独描述时，才从'特殊地形标签池'选择
  坐标: # EJS预留字段
    x: ${公里数} # 以纳维里斯为原点(0,0)，东为正，西为负
    y: ${公里数} # 北为正，南为负
    # 精度要求: ±100km即可。根据交通·旅行距离与时间条目中的数据和地图比例尺填入。
  大致地理: ${一句话地形特征} # 类型: str | 创作: 自由
  人口构成: ${种族比例百分比概述} # 类型: str | 创作: 自由
  环境: ${一句话感官描写} # 类型: str | 创作: 自由
  街景与建筑: ${一句话建筑风貌与街道视觉} # 类型: str | 创作: 自由
  概况: ${一句话功能定位} # 类型: str | 创作: 自由

  相邻节点: # 可扩展数组 | 创作: 自由 | 来源: 从地图和交通条目推导
    - "$__META_EXTENSIBLE__$"
    - 节点名: ${城市/城镇/村庄名}
      距离: ${公里数}km
      路程: ${天数}（${运输方式}）
      路线类型: ${陆路/水路/沿海/山路}
      方位: ${东/西/南/北/东北/西南等}
    # 说明: 仅列直接相连的节点，不列需要中转的。每个节点一条。
    # 城市: 3-8个相邻节点。城镇: 2-4个。村庄: 1-3个。
    # 路程天数按交通·运输方式速度基准条目中最常用的运输方式计算。

  可用运输: ${一句话说明到达和离开这座城市的主要方式} # 类型: str | 创作: 自由
    # 示例: "铁砧隘路陆路（铁壳驮虫转马车），银冠河东支水运可达，无飞行设施"
    # 示例: "翠江水运（顺流至下游各邦），绢稻之路陆路西行，陆行鸟信使站"

  驻扎组织: # 可扩展数组 | 创作: 随机抽取
    - "$__META_EXTENSIBLE__$"
    - ${组织名称}: ${一句话说明该分部的建筑外观或现状} # 类型: str | 创作: 自由
  独有设施: # 可扩展数组 | 创作: 自由
    - "$__META_EXTENSIBLE__$"
    - ${设施名}: ${一句话说明} # 类型: str | 创作: 自由
  当地行业与衍生地点: # 可扩展数组 | 创作: 自由
    - "$__META_EXTENSIBLE__$"
    - ${地点名}: ${一句话说明} # 类型: str | 创作: 自由
  食材可获得性: ${从所属种族食材体系+地理位置+最近贸易路线推导} # 类型: str | 创作: 自由
  消费水平: ${倍率} # 类型: str | 创作: 选择 | 来源: 必须从'消费水平池'选择

  供需网络:
    内部自产: ${本地能自产的资源和食材} # 类型: str | 创作: 自由
      # 示例: "地热温室产土豆和辣椒，矿山产铁矿和煤，本地酿造矮人麦酒"
    外部商队: # 可扩展数组 | 创作: 自由
      - "$__META_EXTENSIBLE__$"
      - 商路: ${商路名称或路线描述}
        来源: ${来自哪座城市/哪个领地}
        运入: ${运入什么资源/食材}
        频率: ${每日/每周/每月/季节性}
      # 说明: 从地理·商路与水系拓扑条目和交通条目推导。
    供需缺口: ${本地完全无法生产、必须靠进口但供应不稳定的东西} # 类型: str | 创作: 自由
      # 示例: "新鲜水果完全依赖南方商队，冬季断供"

  酒馆经营视角:
    客群画像: ${一句话描述谁来吃饭、什么时候来、什么节奏} # 类型: str | 创作: 自由
      # 示例: "矿工和铁匠下工后成群涌入，要快要烫要大份，精致慢餐没人等"
      # 示例: "港口水手来了就走点快餐，本地商人慢慢喝酒谈生意，周末集市日客流翻倍"
      # 示例: "农民熟面孔居多，节奏慢，收获季才热闹，冬天靠本地常客撑着"
    口味需求: ${从主体种族的感官偏移表推导，一句话说明爱什么排斥什么} # 类型: str | 创作: 自由
      # 示例: "矮人口味——重咸重辣不嫌，甜食端出来没人动，矮人觉得刚好的咸度人类端来会皱眉"
      # 示例: "兔族为主——苦味是雷区，甜食是正义，啤酒花麦酒别端上来"
      # 来源: 必须参照种族感官偏移表中对应种族的偏好标签和排斥标签
    口味冲突点: ${如果有混居种族，一句话说明同一道菜不同种族感受的差异和融合机会} # 类型: str | 创作: 自由
      # 示例: "人类军士和犬族商队同桌——犬族嗅觉是人类数十倍，你的菜对犬族是香气轰炸或臭气轰炸，没有中间地带"
      # 示例: "猫族和兔族同桌——猫族对甜完全无感，兔族嗜甜，同一道甜品一个说无聊一个说满足"
      # 纯种族城市: 写"无混居冲突，口味需求单一"
      # 来源: 从人口构成中的混居种族 + 各自的感官偏移表交叉推导
    食材优势: ${这座城市能拿到什么别的同领地城市拿不到的食材} # 类型: str | 创作: 自由
      # 示例: "铁砧隘路终点站，矮人辣椒和香草荚在这里价格最低"
      # 示例: "深界裂隙旁，深界浅层食材日常可得，拉弥亚采集者每天带回新货"
      # 示例: "翡叶香路中转点，精灵香料在这里比内陆便宜三成"
      # 来源: 从地理位置 + 经过的商路 + 独有设施（如深界裂隙）推导
    独特卖点: ${一句话，为什么user应该选这座城市而不是同领地其他城市开酒馆} # 类型: str | 创作: 自由
      # 示例: "全大陆口味最极端的客群——能征服矮人的味觉阈值，回到人类城市做菜就是降维打击"
      # 示例: "铁砧隘路唯一的补给站，北上人类商队和南下矮人驮队都在这里歇脚——天然的融合菜实验场"
      # 示例: "十六个亚种口味全不一样，犬族和猫族坐同一桌你怎么做一盘两边都满意的菜——这就是在联邦开酒馆的终极挑战"
      # 这是最重要的字段，是上面四个字段的交叉结论

  附属定居点: # 仅城市有此字段，城镇和村庄不需要 | 可扩展数组 | 创作: 自由
    - "$__META_EXTENSIBLE__$"
    - 名称: ${附属城镇/村庄名}
      性质: ${城镇/村庄}
      功能: ${一句话功能定位}
      方位与距离: ${方位}约${数字}km
      路程: ${天数}（${运输方式}）
      供给关系: ${为主城提供什么}
    # 说明: 每个附属定居点同时拥有自己的独立条目（按本模板生成），此处仅做附属描述。
    # 城市通常有2-5个附属定居点。

# ═══ 交叉推导原则 ═══

交叉推导原则: 地理位置决定物理约束（什么建得起来、什么运得进来、什么天气要应对），功能类型决定人的活动（谁在这里、为什么来、一天怎么过）。两者交叉时，物理约束改变活动的具体形态——同样的功能在不同地理下长出完全不同的样子。任何地理×任何功能都可能成立，由具体种族和历史背景决定。

# ═══ 基础填写指南 ═══

填写指南:
  工作流:
    阶段1: 定位——确定所属和性质，决定文化底色和规模感
    阶段2: 标签选择——未指定具体标签时，从功能类型池和地理位置核心类型池中各随机选择一个，然后思考这个组合在该种族中如何成立。如果用户指定了具体的功能类型或地理位置，则使用指定的标签
    阶段3: 坐标——根据地图和交通条目中的距离数据，填入以纳维里斯为原点的(x, y)坐标
    阶段4: 骨架填充——大致地理、人口构成、环境、概况，每项一句话
    阶段5: 组织抽取——从驻扎组织池中纯随机抽取（村庄0-1个，城镇1-2个，城市2-3个），无视定居点的功能类型、地理位置和所属种族。用一句话描述该组织分部在这个城市的建筑外观或现状。当随机结果与该种族核心世界观冲突时（如精灵领地抽到蒸汽工会），必须保留该组织并给出冲突如何在当地具体表现的描写
    阶段6: 细节展开——独有设施和行业衍生地点，从世界书中该种族的信仰/文化/产业设定推导，3-6个
    阶段7: 交通网络——填入相邻节点、可用运输方式。从地图坐标和交通条目推导距离与路程，从地理·商路与水系拓扑条目推导经过的商路
    阶段8: 供需网络——内部自产从所属种族食材/资源体系推导，外部商队从经过的商路和相邻节点推导，供需缺口从"本地不产但居民需要"推导
    阶段9: 酒馆经营视角——五个子字段依次推导:
      客群画像: 从功能类型+人口构成推导（工业城=下工工人，商贸城=赶路商人，农业镇=熟面孔农民）
      口味需求: 从种族感官偏移表中主体种族的偏好/排斥标签直接引用
      口味冲突点: 从人口构成中提取混居种族，查各自的感官偏移表，找出同一味觉维度上偏移方向相反的组合（如矮人咸度偏弱+精灵咸度偏强=同一道菜一个说淡一个说咸）
      食材优势: 从地理位置+经过的商路+独有设施推导，重点找"只有这里才有"的食材
      独特卖点: 以上四点的交叉结论，用一句话说明为什么选这里而不是同领地其他城市
    阶段10: 经营数据——食材从所属种族食材体系+地理推导，选消费水平倍率
    阶段11（仅城市）: 附属定居点——根据供需缺口反向推导需要什么样的附属城镇和村庄来填补。每个附属点同时生成独立条目。

  组织与裂隙分离规则:
    如果随机到了冒险者工会，必须将【工会建筑】与【深界裂隙本身】在结构上严格分开描写：
    1. 驻扎组织: 仅描写冒险者工会分部的建筑外观或人员现状（例如：建在火山口边缘的观测站式建筑，或者挤满雇佣兵的石堡）。
    2. 独有设施: 必须单独添加一条【深界裂隙入口】，专门描写裂隙的变异形态。注意：绝大多数裂隙入口连接的是深界极低危险的「林野区/浅表」，普通村民和采药人也能日常进入采集。 裂隙受当地地理影响，主要体现在奇观感和进入方式。

  供需网络推导规则:
    内部自产: 从所属种族的农业/畜牧/矿业/手工业传统 + 地理位置核心类型推导。沿海城市自产海产，内陆平原城市自产谷物，山地城市自产矿石。
    外部商队: 从地理·商路与水系拓扑条目中经过本城的商路推导。不在任何商路上的偏远城市，外部商队频率低且品种少。
    供需缺口: 从"主体种族日常需要但本地地理无法生产"推导。矮人山城缺新鲜蔬果，兽族草原邦缺矿石和金属制品，精灵森林缺畜牧产品。

  酒馆视角推导规则:
    客群画像: 从功能类型+主体种族+性质规模推导。工业城市=下工后涌入的工人；商贸城市=来了就走的商人和赶路车夫；农业村庄=熟面孔、慢节奏、收获季才热闹；军事要塞=军人下班就喝酒不挑剔但量要够。
    口味需求: 必须查种族感官偏移表。直接引用该种族的偏好标签和排斥标签。
    口味冲突点: 从人口构成中识别混居种族。在感官偏移表中找到这些种族之间偏移方向相反的维度——这就是冲突点。同时找到偏移方向相同的维度——这就是融合机会。纯种族城市无此项。
    食材优势: 从以下三个来源交叉推导:
      1. 地理位置——沿海有海产、山地有矿物食材、森林有菌菇草药
      2. 经过的商路——商路沿线城市能以低价获得远方食材
      3. 独有设施——深界裂隙旁有深界食材、地热温室有独家品种、特殊农业有独家产出
      重点是找到"同领地其他城市拿不到的东西"
    独特卖点: 综合以上四项，提炼一句话。这句话的标准是——user读完后能说出"我就去这里"或"我不去这里"，不能是模糊的"这里不错"。

  依赖关系:

    - 功能类型主标签 → 行业衍生地点方向、概况基调、客群画像基调
    - 地理位置核心类型 → 环境感官方向、大致地理、深界裂隙形态、可用运输方式方向

    - 主体种族 + 种族感官偏移表 → 口味需求

  自检清单:

    - 同一种族不同城市的功能类型是否有差异？
    - 独有设施是否来自该种族独有的文化特征？
    - 人口构成是否合理？边境混居多，腹地主体种族多
    - 特殊地形是否真的必要？大部分城市不需要特殊地形标签，只有地形确实无法用核心类型单独描述时才添加
    - 街景描写是否让人能想象出"走在这条街上抬头看到什么"？
    - 驻扎组织与当地的冲突或融合是否有具体描写？不能只写"XX分部"就完事
    - 如果有冒险者工会，是否已将工会建筑（写在驻扎组织）和裂隙形态（写在独有设施）严格分开？
    - 坐标是否已填入？与相邻节点的距离是否与交通条目一致？
    - 相邻节点是否完整？有没有遗漏地图上直接相连的城市？

    - 客群画像是否具体到"谁来、什么时候来、要什么节奏"？不能只写"生意不错"

    - 食材优势是否找到了"只有这里才有"的食材？而不是列举所有食材
    - 独特卖点是否是一句话？读完后能否立刻判断"要去"或"不去"？
    - 附属定居点（如有）是否每个都有独立条目？主城条目里的附属描述是否与独立条目一致？
